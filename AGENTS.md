# AGENTS.md

## Project Overview

Kan is an open-source project management tool (Trello alternative). This is a **Cloudflare Workers fork** ‚Äî the entire stack runs on a single Worker.

- **Frontend**: Next.js 15 static export (`apps/web/out/`), served by the Worker
- **Backend**: Hono on Cloudflare Workers (single Worker serves API + static assets)
- **Database**: Cloudflare D1 (SQLite) via Drizzle ORM
- **File storage**: Cloudflare R2 (avatars, attachments)
- **Auth**: Better Auth
- **API**: tRPC + REST (via trpc-to-openapi)
- **Rate limiting**: Cloudflare KV
- **Email**: React Email + Resend API
- **Monorepo**: pnpm workspaces with Turbo

## Dev Commands

- `pnpm install` ‚Äî install deps
- `pnpm dev:worker` ‚Äî build frontend + start wrangler dev (http://localhost:8787)
- `pnpm build:web` ‚Äî build static frontend
- `pnpm ship` ‚Äî build + deploy to Cloudflare
- `pnpm smoke-test` ‚Äî run smoke tests against localhost
- `pnpm db:migrate:local` ‚Äî apply D1 migrations locally
- `pnpm db:migrate:remote` ‚Äî apply D1 migrations to remote D1
- `pnpm lint:fix` ‚Äî lint and autofix
- `pnpm typecheck` ‚Äî type check all packages
- `pnpm format:fix` ‚Äî format code

### Database Migrations

```bash
cd packages/db && pnpm drizzle-kit generate --name "MigrationName"
pnpm db:migrate:local    # apply to local D1
pnpm db:migrate:remote   # apply to remote D1
```

## Worker Architecture (`apps/worker/`)

The Worker entry point (`src/index.ts`) is a Hono app that mounts route modules:

1. `/api/health` ‚Äî health check
2. Auth routes ‚Äî Better Auth endpoints
3. tRPC routes ‚Äî `/api/trpc/*` (tRPC + OpenAPI REST)
4. Files routes ‚Äî R2 file upload/download (avatars, attachments)
5. Stripe routes ‚Äî webhook handler
6. SPA routes ‚Äî catch-all that serves the static Next.js export from `ASSETS` binding

### Worker Bindings (`Env` in `src/types.ts`)

- `DB` ‚Äî D1 database
- `AVATARS_BUCKET`, `ATTACHMENTS_BUCKET` ‚Äî R2 buckets
- `RATE_LIMIT` ‚Äî KV namespace
- `ASSETS` ‚Äî Workers Assets (serves `apps/web/out/`)
- Secrets: `BETTER_AUTH_SECRET`, `ENCRYPTION_KEY`, `STRIPE_SECRET_KEY`, `STRIPE_WEBHOOK_SECRET`, `EMAIL_API_KEY`, `EMAIL_FROM`

## Code Style

### TypeScript

- Use TypeScript strictly ‚Äî avoid `any` types
- Follow existing patterns for type definitions

### Naming Conventions

- **Files**: kebab-case (e.g., `card-repo.ts`)
- **Components**: PascalCase
- **Functions**: camelCase
- **Constants**: UPPER_SNAKE_CASE
- **Types/Interfaces**: PascalCase

### Logging

- Import from `@kan/logger`: `import { createLogger } from "@kan/logger"`
- Create a module-scoped logger: `const logger = createLogger("module-name")`
- Never use `console.log`

## Database Patterns (`packages/db/`)

- **Schema**: `src/schema/*.ts` ‚Äî Drizzle ORM table definitions for D1 (SQLite)
- **Repositories**: `src/repository/*.repo.ts` ‚Äî all queries go here
- **Migrations**: `migrations/*.sql` ‚Äî generated by `drizzle-kit generate`
- **Soft Deletes**: Use `deletedAt` timestamp; filter with `isNull(table.deletedAt)`
- **Public IDs**: Use 12-character `publicId` for all user-facing entities ‚Äî never expose internal `id` columns
- **Card Indices**: Cards have an `index` field maintained sequentially per list
- **Activity Logging**: Use `card_activity` table to track all card changes
- **Transactions**: Use database transactions for multi-step operations

## API Patterns (`packages/api/`)

- **Routers**: `src/routers/*.ts` ‚Äî tRPC routers
- **Procedures**: `protectedProcedure` for authenticated endpoints, `publicProcedure` for public
- **Validation**: Zod schemas for input validation
- **Error Handling**: `TRPCError` with appropriate codes
- **OpenAPI**: Add OpenAPI metadata for all endpoints (trpc-to-openapi)
- **Authorization**: Always check workspace membership with `assertUserInWorkspace`

## Frontend Patterns (`apps/web/`)

- **Components**: `src/components/`
- **Views**: `src/views/`
- **Pages**: `src/pages/`
- **Hooks**: `src/hooks/`
- **Styling**: Tailwind CSS
- **Data fetching**: tRPC React Query hooks
- **Modals**: `useModal` hook
- **Toasts**: `usePopup` hook
- **i18n**: Lingui ‚Äî use `t` template literal; run `pnpm lingui:extract` after adding strings

## Key Domain Concepts

- **Cards** belong to Lists, which belong to Boards, which belong to Workspaces
- Cards have: title, description, labels, members, checklists, comments, attachments, due dates
- All card changes are tracked in `card_activity`
- Card index management: maintain sequential order on create/move/delete (use transactions)

## Adding a New Feature

1. **Schema**: Update `packages/db/src/schema/`
2. **Migration**: `cd packages/db && pnpm drizzle-kit generate --name "MigrationName"`
3. **Repository**: Add query functions in `packages/db/src/repository/`
4. **API**: Add tRPC router procedures in `packages/api/src/routers/`
5. **Frontend**: Add UI in `apps/web/src/`
6. **i18n**: Add translations, run `pnpm lingui:extract`

## Adding a New Environment Variable

1. `.env.example` ‚Äî add with comment
2. `turbo.json` ‚Äî add to `globalEnv`
3. `apps/worker/src/types.ts` ‚Äî add to `Env` interface
4. `apps/worker/wrangler.toml` ‚Äî add to `[vars]` or document in secrets comment
5. For secrets: set via `wrangler secret put VAR_NAME`

## Security

- Always check workspace membership before operations
- Validate all inputs with Zod
- Never expose internal database IDs ‚Äî use `publicId` externally
- Rate limiting via Cloudflare KV (`RATE_LIMIT` binding)

## Git & Commits

- Conventional commits: `feat:`, `fix:`, `refactor:`, `docs:`, etc.
- Run `pnpm lint` and `pnpm typecheck` before committing
